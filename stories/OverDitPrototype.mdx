# Over dit prototype

Dit prototype van MijnOverheid Zakelijk (MOZa) is gebouwd als statische site met [Eleventy](https://www.11ty.dev/) en gedocumenteerd in Storybook. Het doel is om interactiepatronen en visueel ontwerp te verkennen, zonder een backend.

## Hoe staten worden onthouden

Omdat er geen server of database is, worden gebruikersacties opgeslagen in de **localStorage** van de browser. Dit betekent dat interacties alleen lokaal bestaan en per browser/apparaat verschillen.

### Hoe het werkt

Alle interactielogica staat in **content-interactions.js**, dat op elke pagina wordt geladen via de base-template. Het script:

- Herstelt opgeslagen staten bij het laden van de pagina (checkboxes, radio buttons, verborgen items, gesloten notificaties)
- Slaat wijzigingen direct op bij interactie
- Gebruikt event delegation (`document.addEventListener`) zodat het ook werkt met dynamisch gegenereerde elementen

### Semantiek van de relevantie-keuze

De **Relevant** en **Niet relevant** opties zijn geïmplementeerd als een radio button group binnen een `<fieldset>`:

```html
<fieldset class="relevance-group">
	<legend class="visually-hidden">Is dit relevant voor u?</legend>
	<label class="relevance-option relevance-yes">
		<input type="radio" name="relevance-0" value="relevant" />
		<span class="relevance-icon"></span>
		<span>Relevant</span>
	</label>
	<label class="relevance-option relevance-no">
		<input type="radio" name="relevance-0" value="irrelevant" />
		<span class="relevance-icon"></span>
		<span>Niet relevant</span>
	</label>
</fieldset>
```

- De `<fieldset>` met `<legend>` geeft screenreaders context: “Is dit relevant voor u?”
- Radio buttons zijn natively wederzijds exclusief — de browser handhaaft dit zonder JavaScript
- Toetsenbordnavigatie werkt automatisch via pijltoetsen binnen de groep
- De radio inputs zijn visueel verborgen; de labels zijn gestyled als knoppen met iconen
- Bij “Niet relevant” wordt het item verborgen en schuift een reserve-topic door
- Unieke `name`-attributen worden per groep toegewezen door JavaScript, zodat elke radio group onafhankelijk werkt

### De pagina Bewaarde items

De pagina **Bewaarde items** leest alle `favorite:` en `hidden:` sleutels uit localStorage en rendert deze als volledige, klikbare cards — inclusief titel, URL en beschrijving. Acties op deze pagina (verwijderen, toch relevant markeren) werken ook via localStorage en corrigeren het profiel.

## Berichtenbox

De Berichtenbox simuleert het ophalen van berichten uit meerdere bronnen (overheidsinstanties en gemeentes) via het Federatief Berichtenstelsel. Er is geen backend — alle data wordt gegenereerd tijdens de Eleventy-build en opgeslagen in `window.berichtenboxData`.

### Berichten ophalen

Bij het eerste bezoek aan de Berichtenbox wordt een voortgangsanimatie getoond die simuleert dat berichten worden opgehaald uit meerdere bronnen. De animatie duurt vier seconden en toont het aantal bronnen en gevonden berichten.

### Polling (demo-berichten)

Op de inbox-pagina worden periodiek nieuwe berichten gegenereerd om het "ophalen" van berichten te simuleren. Deze demo-berichten:

- Krijgen een `msg-live-*` ID en worden opgeslagen in localStorage
- Zijn beperkt tot maximaal vijf stuks
- Verschijnen bovenaan de inbox, waarna het onderste bericht naar de volgende pagina verschuift
- Linken naar een generieke detailpagina (`bericht-demo.html`) die het bericht uit de state rendert

De polling-interval is standaard 60 seconden. Voor testen kan `?poll=5` aan de URL worden toegevoegd (minimum 5 seconden).

### Berichtenbox resetten

Op de Berichtenbox-pagina staat een resetlink (`data-berichtenbox-reset`) die alle berichtenbox-state wist en de pagina herlaadt. Handmatig kan het ook via de browserconsole:

```js
localStorage.removeItem("berichtenbox");
```

## Feature flags

Het prototype bevat een ingebouwd feature-flag systeem waarmee pagina's en functionaliteit in- of uitgeschakeld kunnen worden — zonder aparte versies te hoeven bouwen.

### Hoe te gebruiken

Klik op de **Flags**-knop rechtsonder in het prototype. Er opent een paneel met twee groepen:

- **Pagina's** — navigatie-items zoals Berichtenbox, Belastingen en Zakelijk vervoer. Uitschakelen verbergt zowel het navigatie-item als de bijbehorende content op de homepage.
- **Functionaliteit** — interactiemogelijkheden zoals _Delen_ en _Relevantie_. Uitschakelen verbergt de betreffende knoppen in alle actiegroepen.

Feature flags worden opgeslagen in localStorage en blijven staan na herladen. Standaard staan alle features aan.

### Hoe toe te voegen

Voeg `data-feature` en `data-feature-type` attributen toe aan een HTML-element:

```html
{/* Pagina-sectie verbergen */}
<div data-feature="Belastingen" data-feature-type="pagina">...</div>

{/* Functionaliteit verbergen */}
<button data-feature="Delen" data-feature-type="functionaliteit">Deel</button>
```

Het element verschijnt automatisch in het Flags-paneel en kan door de gebruiker aan- en uitgezet worden.

Voor navigatie-items in de sidebar wordt de feature flag ingesteld via het `feature` veld in de data-bestanden (`moza.json` en de subdirectory JSON-bestanden):

```json
{
	"label": "Belastingen",
	"url": "/moza/belastingen/",
	"feature": "Belastingen"
}
```

De template voegt automatisch `data-feature` en `data-feature-type="pagina"` toe aan het `<li>` element.

## localStorage legen

Het Flags-paneel bevat een **localStorage wissen**-knop die alle opgeslagen state in één keer reset en de pagina herlaadt.

Handmatig kan het ook via de browserconsole (F12 of Cmd+Option+I):

```js
localStorage.clear();
```

Herlaad daarna de pagina. Alle bewaarde items, verborgen topics, relevantie-keuzes en gesloten notificaties worden gereset.

Om alleen een specifieke categorie te wissen:

```js
// Alleen bewaarde items wissen
Object.keys(localStorage)
	.filter((k) => k.startsWith("favorite:"))
	.forEach((k) => localStorage.removeItem(k));

// Alleen verborgen items wissen
Object.keys(localStorage)
	.filter((k) => k.startsWith("hidden:"))
	.forEach((k) => localStorage.removeItem(k));

// Alleen relevantie-keuzes wissen
Object.keys(localStorage)
	.filter((k) => k.startsWith("relevant:"))
	.forEach((k) => localStorage.removeItem(k));

// Alleen gesloten notificaties wissen
Object.keys(localStorage)
	.filter((k) => k.startsWith("dismissed:"))
	.forEach((k) => localStorage.removeItem(k));
```
